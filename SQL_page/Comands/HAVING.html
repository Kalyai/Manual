HAVING в SQL

Оператор HAVING в языке SQL применяется после оператора GROUP BY и используется для фильтрации результатов агрегатных функций на основе заданных условий. Он позволяет применять условия к группам строк, сгруппированным по определенному столбцу или столбцам. P.S. Агрегатные данные - это суммирование, подсчет, нахождение среднего, максимума, минимума и другие функции.

Примеры:
  • Использования HAVING после GROUP BY 
    //Таблица sales:
sale_id	product_id	sale_date	 quantity	price
1	    101	        2023-01-01	 2	        50
2	    102	        2023-01-02	 1	        30
3	    101	        2023-01-03	 3	        50
4	    103	        2023-01-04	 5	        20
5	    102	        2023-01-05	 2	        30

SELECT product_id, COUNT(*) AS total_sales
FROM sales
GROUP BY product_id
HAVING total_sales > 1

//Результат:
product_id	total_sales
101	        2
102	        2

//P.S. Столбец total_sales уже является агригатным

    • Выбираем два столбца `product_id` и счетсчик строк (всего строк 5)

    При помощи GROUP BY выбираем строки только с уникальным `product_id` и выбираем количество строк где есть определенный `product_id` (например 101)

    Потом при помощи HAVING создаем условия где total_sales > 1

Используемые команды:

  • SELECT - Выбрать что-то, откуда-то
  • COUNT(...) - Считает количество строк
  • AS - создает псевдоним
  • FROM name - Из таблицы/базы данных с именем name
  • GROUP BY - Группирирует по указанным столбцам
  • HAVING - создает условия после оператора GROUP BY

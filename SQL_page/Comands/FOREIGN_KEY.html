FOREIGN KEY в SQL

FOREIGN KEY (внешний ключ) - это ограничение, которое используется для установления связи между двумя таблицами. Внешний ключ в одной таблице ссылается на первичный ключ или уникальный ключ в другой таблице. Это помогает поддерживать целостность данных, гарантируя, что значения в столбце внешнего ключа соответствуют значениям в столбце первичного или уникального ключа другой таблицы.

Примеры:
  • Пример создания FOREIGN KEY 
    CREATE TABLE People (
	Id INT PRIMARY KEY,
	Name VARCHAR(255) NOT NULL,
	Ip_adress INT FOREIGN KEY REFERENCES Info (Ip_adress)
);

    • Создаем таблицу с `Id`, `Name` и `Ip_adress`

    `Ip_adress` делаем внешним ключом, который зависит от таблицы Info (колонки Ip_adress), а находится в таблице `People`

    Теперь эти две таблицы связанны между собой внешним ключом

Используемые команды:

  • CREATE - Создание таблицы/базы даннных
  • TABLE name - таблица с именем name
